# 0001 — Alcance: el norte es el MVP recomendado

## Contexto
El proyecto tiene un PRD oficial de 35 secciones (`docs/PRD.md`) que describe un producto operativo completo (10-13 meses). El contrato vigente y la capacidad real apuntan a algo más acotado. Ver `docs/propuesta-cliente/01-comparativa-de-alcance.md`.

## Decisión
El **norte de producto** es el **MVP recomendado** (`docs/propuesta-cliente/03-mvp-recomendado.md`), NO el PRD completo. El PRD completo queda como **roadmap** de fases futuras.

## Alternativas consideradas
- Construir el PRD completo — inviable en tiempo/costo para validar.
- Construir lo cotizado original tal cual — desalineado con aprendizajes de mercado.

## Consecuencias
- Foco en diferenciadores (feed vertical, ubicación exacta, CRM básico) sobre commodities.
- El primer hito concreto es aún más acotado: [[0005-demo-cerrada-3-semanas]].
